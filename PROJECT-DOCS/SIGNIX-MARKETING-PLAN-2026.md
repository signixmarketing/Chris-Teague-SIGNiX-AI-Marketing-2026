# SIGNiX Growth Marketing Plan — Executive Summary

**Prepared by:** Chris, Head of Growth and Marketing
**Date:** April 2026
**Status:** Draft — pending executive review and budget approval
**Objective:** Increase monthly transaction volume through three coordinated growth tracks — authentication activation, Flex API partner expansion, and direct demand generation.

---

## Strategic Context

SIGNiX's transaction revenue is constrained by a four-party value chain (buyer → LOS → loan doc vendor → SIGNiX) that limits both activation depth and market width. Closing the transaction gap requires simultaneous action on three fronts: converting existing under-utilizing accounts, expanding the integration partner base, and building direct demand that operates independently of any single channel partner.

## Track 1 — Authentication Activation (Depth)

**Objective:** Increase the percentage of e-sign transactions that include SIGNiX authentications across the TruStage installed base and with net-new customers.

**The problem:** A significant portion of current TruStage-enabled FI customers use SIGNiX for electronic signing with no authentication layer, leaving incremental transaction revenue uncaptured and leaving those customers exposed to signer identity risk.

**Target audience:**
- Tier 1: TruStage-enabled FIs with active e-sign volume and zero or near-zero authentication use
- Tier 2: Net-new FIs (community banks, credit unions) evaluating or upgrading e-sign solutions
- Tier 3: Non-lending FI workflows (wire authorization, beneficiary changes, account opening) within existing accounts

**Tactics:**

| Tactic | Description | Channel |
|--------|-------------|---------|
| **Mike's fraud insights campaign** | Monthly operator-voice email educating FI ops and IT contacts on fraud trends; CTA = workflow review call with Mike | HubSpot email, A/B tested |
| **Direct discovery calls** | Convert email engagements into 30-minute workflow reviews; map top 3 transaction flows per institution | Mike (Client Success) |
| **Authentication self-assessment** | Lightweight guided tool (future): FIs input loan types and current auth methods; output = prioritized risk tier list | Web tool / PDF output |
| **Targeted LinkedIn** | Sponsored content to FI ops, lending, and IT personas; message = authentication as operational risk control, not just a feature | LinkedIn Campaign Manager |
| **Case study development** | 1–2 anonymized examples of FIs that added authentication and the outcome (audit trail, fraud reduction, examiner readiness) | Content asset / sales enablement |

**Key metric:** % of transactions with authentication enabled, measured by account cohort. Baseline required from CTO (pending).

## Track 2 — Flex API Integration Partner Strategy (Width)

**Objective:** Increase the number of software platforms embedding SIGNiX via the Flex API, thereby multiplying the addressable transaction base without depending on a single channel partner.

**The problem:** SIGNiX's best growth historically comes through software integration partners, but Google Ads and email campaigns do not reliably reach software product managers and platform executives. Partnership development requires a different channel mix.

**Target audience:**
- LOS and loan origination platforms (beyond CSI/Hawthorn River)
- Core banking and digital banking platform vendors
- Fintech and regtech software companies serving FIs
- Document management and compliance platforms

**Tactics:**

| Tactic | Description | Channel |
|--------|-------------|---------|
| **Vendor marketplace listings** | Active presence in relevant software marketplaces (nCino, Encompass/ICE, Finastra, Q2, Jack Henry, Temenos) where FI software vendors and buyers discover components | Marketplace / partner portals |
| **Developer documentation** | Invest in clear, searchable Flex API docs and integration guides so ISV developers find SIGNiX organically; reduce time-to-first-transaction | signix.com / developer portal |
| **API-focused Google Ads** | Small allocation (~$500–700/month) targeting developer and technical terms: `e-signature API financial services`, `loan document signing API`, `white label e-sign API` | Google Search |
| **LinkedIn ISV targeting** | Sponsored outreach to product managers, VP Engineering, and CTOs at LOS and fintech companies; message = Flex API capabilities + regulated industry compliance | LinkedIn Campaign Manager |
| **Analyst briefings** | Brief Info-Tech Research Group (secured at no cost) and target Gartner/Forrester as budget allows; ensure SIGNiX is named in e-sign API evaluations | Analyst relations |
| **Conference presence** | Selective attendance at FI technology conferences (e.g. Jack Henry Connect, Finastra Universe, regional CUNA/NAFCU events) where software vendors and FI buyers overlap | Events |

**Key metric:** Number of new Flex API integration agreements signed per quarter; number of ISV discovery conversations initiated per month.

## Positioning Principles (apply across all tracks)

- **Regulated industry focus:** Financial services, banking, credit unions, lending — not generic horizontal messaging.
- **Authentication differentiation:** PKI-based digital signatures + configurable signer authentication separates SIGNiX from DocuSign/Adobe's default click-to-sign.
- **Partner-first language:** "Helping you get more value from what you already have" — not "you're doing it wrong."
- **Fear opens doors; clarity closes them:** Fraud education starts the conversation; workflow specificity and audit-trail proof convert it.
- **RON as a cross-sell wedge:** Proven lead-gen channel; use it to introduce e-sign and authentication to net-new prospects.
