The Certificate Chain
A SIGNiX signature traces to GlobalSign AATL with a signer-level certificate. The signer's IP address, device, and transaction ID are embedded directly in the PDF. A standard platform uses a server certificate ("DS Technical Operations" or similar). That certificate belongs to the vendor. It does not name the signer.
ByteRange Tamper Coverage
The ByteRange is the portion of the PDF covered by the cryptographic hash. SIGNiX signatures cover 100% of the file. If one character changes after signing, the hash fails and tampering is detected. The Inspector shows the exact byte coverage — live, on any document.
The PKI-Signed Audit Trail
SIGNiX produces two PKI-signed files per transaction: the signed document and the audit trail. Both are cryptographically anchored to GlobalSign. Standard platforms use a plain PDF for the Certificate of Completion. No PKI signature. No ByteRange protection. That record can be challenged in a dispute.
